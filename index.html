Why Python?

Static virtual machines make poor runtimes for dynamically-typed languages. They constrain the language to their view of the world and limit the options available to language implementors. We're writing an implementation of Clojure that runs on the Python VM to prove it. With a proper dynamic JIT (like PyPy), a version of Clojure running on a dynamic VM can outperform its JVM and CLR counterparts.

Also, there are many Python libraries like PySide (Qt GUI), numpy, scipy, and stackless that do not have JVM counterparts, or the Python implementations are easier to use and learn. Clojure-Py integrates tightly with the Python VM and is able to use all these libraries.

Installation

To install version 0.1.0 from PyPI:

pip install clojure-py

Or from source:

git clone http://github.com/halgari/clojure-py.git
cd clojure-py
python setup.py install

Run with

clojurepy

Or, if the source is available

python clojure.py

Run tests with nose:

nosetests

Basic concepts

Viewing Clojure-Py's core.clj is probably the best way to get a feeling of what is possible and how Clojure-Py works.

Specifically, you might notice...

  • Python builtins are available under the py/ namespace. Actual Python bytecodes can be injected with py.bytecodes/OP.
  • Clojure-Py implements ClojureScript's property vs. calling method design: 
    (.__name__ module) ; same as module.__name__() in Python
(.-__name__ module) ; same as module.__name__ in Python

How can I help?

Find a need and fill it! Play around with Clojure-Py, start porting your favorite Clojure lib, and see what's missing. Feel free to join our mailing list. We normally send out a message when we plan to work on something from the issues list or something that came up in the mailing list. There are quite a few functions from clojure.core that need porting. Drop by and let us know what your're interested in. We'll be glad to help!
